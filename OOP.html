Object-Oriented Programming (OOP)

This page documents my work for the Object-Oriented Programming module. It brings together weekly artefacts (code, UML, testing), short notes on what I learned each week, and the final project delivery and evaluation.

Main goals for the module

  1. Explore the development of OOP using the Python programming language.
  2. Gain practical experience designing object-oriented programs with the Unified Modeling Language (UML).
  3. Recognise appropriate data structures to retain and manipulate the data in question.
  4. Examine efficient ways to search for and retrieve data.
  5. Appreciate the role of design patterns in creating maintainable software.

01 UNIT: An Introduction to Python and the OOP Paradigm

Required Reading

Fundamentals of Python and the four pillars of OOP: abstraction, encapsulation, inheritance, and polymorphism. Defined basic classes and explored core data types.

Outcomes: describe key OOP features; define a Python class; assign and use Python data types.

03 UNIT: Unified Modeling Language (UML)

Required Reading

Core UML syntax and commonly used diagrams across the SDLC: class, sequence, activity, and use-case diagrams.
